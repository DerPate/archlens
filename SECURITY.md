# Security Policy

## Reporting a Vulnerability

Please do not open a public issue for suspected security vulnerabilities.

Use GitHub private vulnerability reporting if it is enabled for the repository. If it is not enabled yet, contact the maintainer privately and include:

- A description of the vulnerability.
- Steps to reproduce or a proof of concept.
- The affected version or commit.
- Any known mitigations.

## Supported Versions

Until the project publishes stable releases, security fixes are made against the default branch.
